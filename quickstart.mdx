This guide will get you from zero to a running Flowbaker executor in under 5 minutes.

## What you'll need

<Check>Go 1.25.0 or higher (if building from source)</Check>
<Check>A Flowbaker workspace account</Check>
<Check>Network access to api.flowbaker.io</Check>

## Installation

<Steps>
  <Step title="Download the binary">
    Download the latest release for your platform:
    
    <CodeGroup>
    ```bash Linux (amd64)
    curl -L https://github.com/flowbaker/flowbaker/releases/latest/download/flowbaker-linux-amd64 -o flowbaker
    chmod +x flowbaker
    ```
    
    ```bash macOS (Apple Silicon)
    curl -L https://github.com/flowbaker/flowbaker/releases/latest/download/flowbaker-darwin-arm64 -o flowbaker
    chmod +x flowbaker
    ```
    
    ```bash macOS (Intel)
    curl -L https://github.com/flowbaker/flowbaker/releases/latest/download/flowbaker-darwin-amd64 -o flowbaker
    chmod +x flowbaker
    ```
    </CodeGroup>
    
    Or build from source:
    
    ```bash
    git clone https://github.com/flowbaker/flowbaker.git
    cd flowbaker
    go build -o flowbaker cmd/main.go
    ```
  </Step>
  
  <Step title="Start the executor">
    Run the executor binary:
    
    ```bash
    ./flowbaker start
    ```
    
    On first run, you'll see an interactive setup wizard that guides you through:
    
    - Creating your executor ID
    - Connecting to your Flowbaker workspace  
    - Generating authentication keys
    
    <Info>The executor starts a health check server on port 8081 and then connects to your workspace.</Info>
  </Step>
  
  <Step title="Complete the setup wizard">
    Follow the prompts to configure your executor:
    
    ```
    Welcome to Flowbaker Executor Setup!
    
    ? Enter your workspace URL: https://app.flowbaker.io
    ? Enter your workspace passcode: ********
    ? Enter a name for this executor: My Executor
    
    ✓ Executor registered successfully
    ✓ Configuration saved to ~/.flowbaker/config.json
    ✓ Starting executor...
    ```
  </Step>
  
  <Step title="Verify it's running">
    Check the executor status:
    
    ```bash
    ./flowbaker status
    ```
    
    Expected output:
    
    <Accordion title="Example status output">
    ```
    Executor Status: Running
    Executor ID: exec_xxxxxxxxxxxxx
    Executor Name: My Executor
    
    Workspaces:
      - your-workspace (https://your-workspace.flowbaker.io)
    
    API URL: https://api.flowbaker.io
    Last Connected: 2s ago
    ```
    </Accordion>
    
    You can also check the health endpoint:
    
    ```bash
    curl http://localhost:8081/health
    ```
  </Step>
</Steps>

## Run your first workflow

Once your executor is running, you can assign workflows to it from your Flowbaker dashboard:

1. **Create a workflow** in your Flowbaker workspace
2. **Add nodes** - Start with a simple webhook trigger and HTTP action
3. **Assign to executor** - Select your executor from the dropdown
4. **Activate the workflow** - Toggle it on
5. **Test it** - Send a request to the webhook URL

<Tip>Use `./flowbaker start --debug` for verbose logging when testing workflows.</Tip>

## Workspace management

### List workspaces

```bash
./flowbaker workspaces list
```

This shows all workspaces your executor is registered with.

### Assign via dashboard

To assign your executor to additional workspaces:

1. Go to your Flowbaker dashboard
2. Navigate to **Settings** → **Executors**
3. Find your executor by ID
4. Add it to the desired workspace

## Troubleshooting

<AccordionGroup>
  <Accordion title="Executor won't connect">
    **Check API connectivity:**
    
    ```bash
    curl https://api.flowbaker.io/health
    ```
    
    **Verify your passcode and workspace URL are correct.**
    
    **Check firewall rules** - Ensure outbound HTTPS (port 443) is allowed.
  </Accordion>
  
  <Accordion title="Port 8081 already in use">
    Another service is using port 8081. Either:
    
    - Stop the conflicting service
    - Use a different port (not currently supported in CLI)
    - Deploy with Docker and map to a different port
  </Accordion>
  
  <Accordion title="Configuration not saved">
    Ensure you have write permissions to `~/.flowbaker/`:
    
    ```bash
    mkdir -p ~/.flowbaker
    chmod 700 ~/.flowbaker
    ```
  </Accordion>
  
  <Accordion title="Workflows not executing">
    **Check executor status:**
    
    ```bash
    ./flowbaker status
    ```
    
    **Verify workflow is active** in your dashboard.
    
    **Check executor assignment** - Make sure the workflow is assigned to your executor.
    
    **Enable debug logging:**
    
    ```bash
    ./flowbaker start --debug
    ```
  </Accordion>
</AccordionGroup>
