Providers are implementations of the `LanguageModel` interface that connect to different LLM APIs.

## LanguageModel Interface

From `pkg/ai-sdk/provider/provider.go:11-27`:

```go
type LanguageModel interface {
    // Generate produces a complete response (blocking)
    Generate(ctx context.Context, req GenerateRequest) (*types.GenerateResponse, error)
    
    // Stream creates a streaming request
    Stream(ctx context.Context, req GenerateRequest) (*ProviderStream, error)
    
    // ID returns the unique identifier for this model
    ID() string
    
    // Capabilities returns the capabilities of this model
    Capabilities() Capabilities
    
    // ProviderName returns provider name
    ProviderName() string
}
```

## OpenAI Provider

From `pkg/ai-sdk/provider/openai/openai.go`:

### Creating an OpenAI Provider

```go
import "github.com/flowbaker/flowbaker/pkg/ai-sdk/provider/openai"

// Basic setup
model := openai.New(apiKey, "gpt-4o")

// With custom base URL (for compatible APIs like Groq)
model := openai.New(apiKey, "mixtral-8x7b-32768",
```

```go

model.SetRequestSettings(openai.RequestSettings{
    Model:            "gpt-4o",
    Temperature:      0.7,
    MaxTokens:        2000,
    TopP:             0.9,
    FrequencyPenalty: 0.5,
    PresencePenalty:  0.5,
    Stop:             []string{"\n\n"},
})
```

```go
stream, err := model.Stream(ctx, provider.GenerateRequest{

    Tools:    tools,
})

    log.Fatal(err)
}

for event := range stream.Events {
    // Process events
}

if err := stream.Err(); err != nil {
    log.Fatal(err)
}
```

## Best Practices

1. **Choose the right model** for your use case (speed vs. capability)
2. **Set appropriate token limits** to control costs
3. **Use streaming** for better user experience
4. **Handle errors** from both provider initialization and API calls
5. **Monitor token usage** with the Usage field
6. **Set temperature carefully** (lower = more deterministic, higher = more creative)
7. **Use stop sequences** to control output format
8. **Consider context windows** when managing conversation history
