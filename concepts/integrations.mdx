## Overview

Integrations in Flowbaker enable workflows to interact with external services, perform computations, and process data. Each integration provides a set of triggers and actions that can be used in workflow nodes.

## Integration Architecture

The integration system is built on a modular architecture:

```go
type Integration struct {
    ID                   IntegrationType
    Name                 string
    Description          string
    CredentialProperties []NodeProperty
    Actions              []IntegrationAction
    Triggers             []IntegrationTrigger
    EmbeddingModels      []IntegrationEmbeddingModel
    IsGroup              bool
    CanTestConnection    bool
    IsCredentialOptional bool
}
```

Location: `pkg/domain/integration.go:86`

### Integration Components

Each integration can provide:

1. **Creator**: Creates integration instances for execution
2. **Poller**: Handles polling-based triggers
3. **Connection Tester**: Validates credentials
4. **HTTP Client Provider**: Provides authenticated HTTP clients

## Integration Actions

Actions are operations that integrations can perform:

```go
type IntegrationAction struct {
    ID                            string
    ActionType                    IntegrationActionType
    Name                          string
    Description                   string
    Properties                    []NodeProperty
    HandlesByContext              map[ActionUsageContext]ContextHandles
    SupportedContexts             []ActionUsageContext
    CombinedContexts              []ActionUsageContext
    IsNonAvailableForDefaultOAuth bool
    Decoration                    NodeDecoration
}
```

Location: `pkg/domain/integration.go:178`

### Action Properties

- **ActionType**: Unique identifier for the action (e.g., `http_request`, `send_message`)
- **Properties**: Configurable properties for the action
- **HandlesByContext**: Input/output handles for different usage contexts
- **SupportedContexts**: Where this action can be used (workflow, tool, etc.)

### Example: HTTP Request Action

```go
IntegrationAction{
    ID:         "http-request",

    Name:       "HTTP Request",
    Description: "Make an HTTP request to any endpoint",

            Key:      "method",
            Name:     "Method",

            Required: true,
            Options: []NodePropertyOption{
                {Label: "GET", Value: "GET"},
                {Label: "POST", Value: "POST"},

            Key:         "url",
```

## Integration Execution

When a node executes an integration:

### 1. Select Integration Creator

```go
integrationCreator, err := integrationSelector.SelectCreator(ctx, 
    domain.SelectIntegrationParams{
        IntegrationType: node.IntegrationType,
    })
```

### 2. Create Integration Instance

```go
integrationExecutor, err := integrationCreator.CreateIntegration(ctx, 
    domain.CreateIntegrationParams{
        WorkspaceID:  workspaceID,
        CredentialID: credentialID,
    })
```

### 3. Execute Integration

```go
output, err := integrationExecutor.Execute(ctx, domain.IntegrationInput{
    NodeID:           node.ID,
    PayloadByInputID: payloadByInputID,
    IntegrationParams: domain.IntegrationParams{
        Settings: node.IntegrationSettings,
    },
    ActionType: actionType,
})
```

## Creating Custom Integrations

To add a new integration:

### 1. Define Integration Structure

```go
package myintegration

import "github.com/flowbaker/flowbaker/pkg/domain"

type MyIntegrationCreator struct {
    deps domain.IntegrationDeps
}

func NewMyIntegrationCreator(deps domain.IntegrationDeps) domain.IntegrationCreator {
    return &MyIntegrationCreator{deps: deps}
}
```

### 2. Implement Creator Interface

```go
func (c *MyIntegrationCreator) CreateIntegration(
    ctx context.Context, 
    params domain.CreateIntegrationParams,
) (domain.IntegrationExecutor, error) {
    return &MyIntegration{
        credentialID: params.CredentialID,
        workspaceID:  params.WorkspaceID,
        deps:         c.deps,
    }, nil
}
```

### 3. Implement Executor Interface

```go
type MyIntegration struct {
    credentialID string
    workspaceID  string
    deps         domain.IntegrationDeps
}

func (i *MyIntegration) Execute(
    ctx context.Context, 
    input domain.IntegrationInput,
) (domain.IntegrationOutput, error) {
    // Get credentials
    cred, err := i.deps.ExecutorCredentialManager.GetFullCredential(ctx, i.credentialID)
    
    // Bind parameters from settings
    var params MyActionParams
    err = i.deps.ParameterBinder.BindToStruct(ctx, input, &params, nil)
    
    // Execute integration logic
    result := i.performAction(params)
    
    // Return output
    return domain.IntegrationOutput{
        ResultJSONByOutputID: []domain.Payload{resultJSON},
    }, nil
}
```

### 4. Register Integration

Add to `internal/initialization/integrations.go`:

```go
{
    IntegrationType:     domain.IntegrationType_MyIntegration,
    NewCreator:          myintegration.NewMyIntegrationCreator,
    NewConnectionTester: myintegration.NewMyIntegrationConnectionTester,
}
```

## Best Practices

<Check>
**Do's**
- Implement connection testing for integrations with credentials
- Provide clear property descriptions
- Support expression evaluation in properties
- Handle errors gracefully
- Use appropriate handle types (success/error)
</Check>

<Warning>
**Don'ts**
- Don't expose credentials in logs
- Don't assume credentials are always present
- Don't block indefinitely in Execute()
- Don't ignore context cancellation
</Warning>
